## Mandatory Start Behavior
Before making any file changes:
1. Read the required files listed above.
2. Determine whether the task follows `Skeleton Build Mode` or `Modernization Mode`.
3. If the mode is not explicit, ask the user which path is intended before planning file edits.
4. Tell the user you have read the project rules.
5. Briefly restate the rules that apply to the current request.
6. State which files you expect to change and why.
7. Wait for explicit user approval before editing files.

Without explicit user approval, you may only:
- read files,
- inspect code,
- run non-mutating diagnostics,
- explain findings.

## Mode Definitions
- `Skeleton Build Mode` = report is built from sample/image/requirements and working data sources
- `Modernization Mode` = working `index.html`, `hc-report.css`, and `script.js` already exist and must be improved

## Required Response After Reading
After reading the required files, do not jump straight into editing.
Respond to the user in this format:
- confirm that you read the project rules,
- state whether the task is in `Skeleton Build Mode` or `Modernization Mode` and ask if unclear,
- briefly restate the key rules you will follow,
- list the files you expect to change,
- say that you are ready,
- wait for approval.
